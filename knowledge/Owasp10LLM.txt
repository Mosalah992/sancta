LLM01: Prompt Injection – Manipulating an LLM through crafty inputs to cause unintended actions.LLM02: Insecure Output Handling – Downstream systems blindly accepting LLM output without scrutiny.LLM03: Training Data Poisoning – Manipulating training data or fine-tuning to introduce backdoors or biases.LLM04: Model Denial of Service – Causing resource-heavy operations to degrade service or increase costs.LLM05: Supply Chain Vulnerabilities – Compromises in the application lifecycle through third-party components or services.LLM06: Sensitive Information Disclosure – Inadvertent revelation of confidential data in model responses.LLM07: Insecure Plugin Design – Insecure inputs or insufficient access controls in LLM plugins.LLM08: Excessive Agency – Granting the LLM-based system too much functionality, permission, or autonomy.LLM09: Overreliance – Depending too heavily on LLM output without oversight, leading to misinformation or security issues.LLM10: Model Theft – Unauthorized access, copying, or exfiltration of proprietary models.LLM01: PROMPT INJECTION Attackers manipulate the LLM using crafted inputs to execute their own intentions, either directly (overwriting system prompts) or indirectly (via external sources).Examples: Direct injection to overwrite instructions; indirect injection via a webpage the LLM is summarizing.Prevention: Enforce privilege control on backend access, keep humans in the loop for extensible functionality, and segregate external content from user prompts.LLM02: INSECURE OUTPUT HANDLING Occurs when LLM output is accepted without validation, potentially leading to XSS, CSRF, SSRF, or remote code execution (RCE).Examples: Output entered directly into a system shell; LLM-generated JavaScript causing XSS in a browser.Prevention: Apply input validation on responses sent to backend functions and encode output before returning it to users.LLM03: TRAINING DATA POISONING Manipulating the data or fine-tuning process to compromise security, effectiveness, or ethical behavior.Examples: Creating malicious documents for training; training on falsified or unverified data.Prevention: Verify the legitimacy of all data sources; use strict vetting and input filters for specific training categories.LLM04: MODEL DENIAL OF SERVICE Attackers consume excessive resources (computation, memory, etc.), leading to service degradation or high costs.Examples: Queries leading to high-volume task generation; sending input that exceeds the context window.Prevention: Implement input validation/sanitization; enforce API rate limits and cap resources per request.LLM05: SUPPLY CHAIN VULNERABILITIES Vulnerabilities stemming from outdated software, pre-trained models, or insecure third-party plugins.Examples: Using outdated Python packages; fine-tuning with a compromised pre-trained model.Prevention: Vet data sources; use trusted plugins; apply MLOps best practices and maintain a patching policy.LLM06: SENSITIVE INFORMATION DISCLOSURE Inadvertent disclosure of proprietary algorithms, confidential data, or PII through model responses.Examples: Incomplete filtering of sensitive data; overfitting/memorizing sensitive data during training.Prevention: Use data sanitization and scrubbing; implement robust input/output validation; apply the rule of least privilege.LLM07: INSECURE PLUGIN DESIGN Plugins accepting malicious requests due to insufficient access control or improper input validation.Examples: Plugins accepting raw SQL/code; authentication without explicit authorization for specific actions.Prevention: Enforce strict parameterized inputs; require manual user authorization for sensitive plugin actions.LLM08: EXCESSIVE AGENCY Granting LLMs too much autonomy or high-privileged access to downstream systems.Examples: An LLM agent accessing unnecessary plugin functions; a plugin with high-privileged identity on other systems.Prevention: Limit tools/functions the LLM can call; require human approval for all actions; log and monitor activity.LLM09: OVERRELIANCE Accepting LLM-generated content as fact without verification, which can lead to misinformation or security breaches.Prevention: Maintain human oversight and verify LLM outputs against trusted sources before use.LLM10: MODEL THEFT Unauthorized access or exfiltration of proprietary LLM models, leading to economic and competitive loss.Prevention: Implement strict access controls, monitor for exfiltration, and protect model weights.
